GoldSun

A stripped-down Mac browser built for speed, security, and a calmer native feel.

Native Swift UI HTTPS first Native WebKit UA

Security without browser bloat.

GoldSun keeps the surface area small while tightening the defaults that matter most in daily browsing.

HTTPS first

HTTP navigations upgrade to HTTPS by default, with an optional strict mode for sites that should never fall back.

Tracker cleanup

Known tracking query parameters are removed before navigation, including common ad click and campaign identifiers.

Native content blocking

The WebKit development backend installs a compact rule list for common ads, trackers, and strict-mode noise.

Keychain passwords

Saved logins use macOS Keychain storage, exact-origin autofill, native save prompts, and browser CSV import/export for migration.

Fast by design

GoldSun opens to an offline start page, keeps the UI native, and avoids heavy dependencies in the shell. The result is a browser that feels immediate before the Chromium backend work even lands.

Polished where it counts

Version 0.2.14 keeps the new Chromium/CEF browsing surface prompt-free: no macOS permission dialogs during development builds, and no Keychain interruptions while browsing.

Version 0.2.14

The prerelease package installs GoldSun into Applications with a Dock/Finder icon, the refreshed mountain start page, native bookmarks with browser import/export, built-in bookmark, history, download, and password pages, Keychain-backed password storage with save prompts, default-browser registration metadata, signed-build passkey readiness, top tab bar browsing, YouTube-compatible fullscreen video, a default-browser handoff for sites that block embedded sign-in, a downloads popover, auto update checks that quit GoldSun before installation, and security-first defaults. This version adds the first Chromium/CEF proof-of-life engine for local builds; release artifacts remain WebKit-only until CEF packaging lands in CI. Chrome Web Store installation controls are hidden until a real Chromium extension runtime is available. It is unsigned until Developer ID signing and notarization are configured.
